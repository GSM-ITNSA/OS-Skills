
### Fault Tolerant 
#### 장애 허용성이란?
```
시스템내의 어느 한 부품 또는 어느 한 모듈에 Fault(장애)가 발생하더라도 시스템운영에 전혀 지장을 주지 않도록 설계된 컴퓨터 시스템

Fault Tolerant 컴퓨터 시스템내의 대부분의 부품 또는 모듈을 시스템이 운영되거나 응용 프로그램이 돌아가는 동안에라도 빼내거나 장착할 수가 있다

따라서 고장난 모듈을 온라인 상태에서 사용자가 전혀 모르게 교환이 가능하며, 어떠한 Fault가 발생하더라도 아무런 데이터의 손실 및 파괴없이 시스템은 정상동작을 계속한다.
```
#### 단계
- 장애 감지(Fault Detection)
    ```
    Fault Detection은 주로 하드웨어로 구성된 비교기(Compare Logic)을 통하여 이루어진다.
    
    시스템내에서 Fault가 발생되면 해당 모듈 또는 시스템은 Fault 상태로 들어가게 된다.
    
    OS는 각 모든 하드웨어 모듈들의 상태를 분석하여 어느 모듈이 Fualt를 유발시켰는가를 
    분석하여 알아낸다.
    ```
- 장애 진단(Fault Diagnosis)
    ```
    Fault 는 일시적이거나 영구적일 수 있다. 만약 Fault가 영구적이라면 그 모듈을 시스템 구성에서 제거한다.

    만약 Fault가 일시적이라면 즉, Fault가 발생하여 자기 진단을 수행한 결과 아무런 문제가없다고 판단되면, 시스템은 일단 그 Fualt를 일시적이라 인식하고 모든 동작을 계속 수행한다.

    그러나 일시적 장애가 일정시간 동안에 정의된 수를 넘게 되면 OS는 이 또한 시스템 구성에서 제거한다.
    ```
- 장애 복구(Fault Recovery)
    ```
    Fault를 유발한 모듈을 시스템에서 제거하여 시스템을 재구성하면서 이루어진다.

    Fault Tolerance의 중요한 기능 중 하나는 데이터를 항상 2개 복사하도록 해놓음으로써 유지시킨다는 점이다

    즉 CPU가 데이터 쓰기를 할 경우 항상 2개의 메모리에 저장시켜 놓는 "메모리 Shadowing"을 수행한다.

    어느 한 CPU(또는 CPU 모듈)가 메모리에 있는 Process를 수행하는 과정에서 이상이 생기더라도 또 다른 메모리에 있는 데이터를 다른 CPU(또는 CPU 모듈)가 수행하게 함으로써 시스템에 영향을 전혀 주지 않고 Fault Tolerance를 유지할 수 있다.
    ```
#### 장점
```
시스템이 하나 이상의 구성 요소의 장애에도 계속 작동할 수 있으므로 서비스 중산 시간을 최소화하고 사용자에게 높은 신뢰성을 제공합니다.
```
#### 단점
```
고가용성을 유지하려면 추가 리소스와 비용이 필요하며, 구현 및 유지 관리가 복잡할 수 있다.
```
